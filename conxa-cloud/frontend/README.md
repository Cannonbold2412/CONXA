# Skill Review UI

Skill compiler single-flow editor: record actions, compile JSON, human-in-the-loop step editing, validation, screenshots, suggestions, and final recompile.

**Stack:** Next.js App Router, React 19, Clerk, TanStack Query, Zustand, [Tailwind CSS 4](https://tailwindcss.com) + [shadcn/ui](https://ui.shadcn.com) (Radix Nova), [Sonner](https://sonner.emilkowal.ski) toasts, Geist font.

**Design profile (devtool / workbench):** dark shell (`<html class="dark">`), three-column review grid on desktop, stacked on small viewports, borders and `muted` surfaces for density. [UI/UX Pro Max](https://github.com/nextlevelbuilder/ui-ux-pro-max-skill) (optional) — run `npx uipro init --ai cursor` in the repo to add searchable palettes/typography guidance for future pages.

**21st.dev Magic (optional):** With the Magic MCP in Cursor, use the component builder for new sub-patterns (e.g. timeline, log strip); align generated code with this stack (`@/` imports, Tailwind, `components/ui/*`). A reference pattern lives in `ValidationReportPanel` (collapsible JSON readout).

**Resilience:** [ErrorBoundary](src/components/ErrorBoundary.tsx) wraps the client app providers with a recovery affordance. Prefer visible focus (`focus-visible:ring` on shadcn controls) and `aria-live` on recording status in `HomePage` / `HumanEditPage`. Global `prefers-reduced-motion` is respected for scroll behavior in [index.css](src/index.css).

## Run

1. **API** (repo root, port 8000 by default): `uvicorn app.main:app --reload --host 127.0.0.1 --port 8000`
2. **Env vars**: create `frontend/.env.local` and set:
   - `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY`
   - `CLERK_SECRET_KEY`
   - `NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in`
   - `NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up`
   - `API_ORIGIN`
   - `CONXA_API_PROXY_SECRET`
3. **This app**: `npm install` then `npm run dev`.
4. **Open** `http://127.0.0.1:3000/` and run the flow in this order:
   - Start recording from a URL.
   - Close/stop recording to compile.
   - Edit generated steps in the same screen.
   - Recompile from session for the final package.

The Next route handler under `app/api/v1/[...path]/route.ts` proxies `/api/v1/*` requests to `API_ORIGIN` and forwards the current Clerk session token.
It also forwards a trusted server-only identity header set when `CONXA_API_PROXY_SECRET` matches the backend `SKILL_API_PROXY_SHARED_SECRET`; this lets the backend resolve the active Clerk organization workspace reliably.

### API base URL (production)

- Set `API_ORIGIN` to your backend origin **without** a trailing slash, for example `https://api.example.com`.
- Set `CONXA_API_PROXY_SECRET` to the same high-entropy value configured as `SKILL_API_PROXY_SHARED_SECRET` on the backend. Do not prefix it with `NEXT_PUBLIC_`.
- The frontend always uses same-origin `/api/v1/*` calls; the Next handler forwards them upstream.

## Source layout (high level)

| `src` area | Purpose |
|------------|--------|
| `api/` | API calls and data loading |
| `components/` | UI for workflow, steps, validation, screenshots, etc. |
| `store/` | Zustand editor state |
| `types/` | Shared TypeScript types |

`.next/` is the generated production/dev build output; `node_modules/` is dependencies. Both are generated by the toolchain.

## Build (production)
